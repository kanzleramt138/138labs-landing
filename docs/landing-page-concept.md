# Landing Page Concept

## Goal

The landing page should move from a branded placeholder toward a lightweight business entry point:

- clarify what 138Labs does
- show who the work is for
- reduce friction to contact
- stay visually sharp and technically simple

## Current state

The current page already has:

- strong visual identity
- clear personal branding
- direct contact links
- low operational complexity

What it lacks is business framing. A visitor currently sees style before offer.

## Recommended page structure

Keep the page as a single static page and expand it in sections.

### 1. Hero

Keep:

- logo
- strong headline
- primary contact actions

Improve:

- state the business value more explicitly
- add one primary CTA and one secondary CTA

Suggested direction:

- Headline: practical AI, automation, and software systems
- Subline: build internal tools, workflows, and custom software without agency overhead

### 2. Services

Add a section with 3 focused cards:

- AI tooling
- workflow automation
- custom software / internal tools

Each card should answer:

- what gets built
- for whom
- what outcome it creates

### 3. Use cases

Add a section with concrete examples:

- lead handling and CRM automation
- internal dashboards and ops tools
- LLM-assisted support or research workflows
- integration-heavy business process automation

This section matters because it helps prospects self-identify quickly.

### 4. Delivery model

Add a short section explaining how you work:

- fast scoping
- direct founder/developer collaboration
- pragmatic delivery
- iterative releases

This differentiates you more than generic agency language.

### 5. Proof / credibility

If you have references, add one of:

- selected projects
- capabilities matrix
- short case-study snippets
- stack / tooling credibility block

If references are not ready yet, add a "currently building with" style section instead.

### 6. Contact

End with one clear conversion area:

- email
- LinkedIn
- optional booking link later

## Design direction

The current neon aesthetic is worth keeping, but it should support content rather than dominate it.

Recommended direction:

- keep the dark branded hero
- reduce visual intensity in lower sections
- use section contrast to improve readability
- introduce a more editorial layout below the fold

## Technical direction

Do not introduce a framework yet unless one of these becomes true:

- you need CMS-like content maintenance
- you add dynamic forms or analytics-heavy experiments
- the page grows into a broader marketing site

For now, plain HTML/CSS is the correct tradeoff.
